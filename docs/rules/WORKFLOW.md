# Development Workflow (개발 프로세스)

이 문서는 프로젝트의 작업 관리 및 진행 방식을 정의합니다.

## 1. 문서 기반 작업 관리

작업은 다음 세 가지 마크다운 파일을 통해 관리됩니다.

- **`tasks/issues/`**: 발견된 모든 버그, 보안 위험, 기술 부채를 개별 파일로 기록합니다. (예: `001_AUTH_LOOP.md`)
- **`tasks/todo/`**: `issues/`에서 도출된 해결책이나 새로 추가할 기능 등 '할 일'을 개별 파일로 기록합니다.
- **`tasks/HISTORY.md`**: 완료된 작업의 이력을 통합 관리하여 프로젝트의 발전 과정을 추적합니다.

## 2. 작업 흐름 (Task Flow)

1. **문제 발견**: 새로운 이슈가 발견되면 `tasks/issues/` 폴더에 새 마크다운 파일을 생성합니다. 이때 파일 이름 앞에 순차적인 번호를 부여합니다 (예: `003_NEW_ISSUE.md`).
2. **계획 수립**: 이슈 해결 방법이나 새 기능 구현 계획을 `tasks/todo/` 폴더에 생성합니다. 동일하게 번호를 부여합니다.
3. **구현 및 검증**: 코드를 작성하고 테스트를 완료합니다.
4. **완료 처리**: 작업이 끝나면 `tasks/todo/`의 파일을 삭제하거나 내용을 `tasks/HISTORY.md`의 연대기에 통합하여 업데이트합니다.

## 3. 문서 업데이트 규칙

- 모든 작업이 완료된 후에는 관련 문서를 최신 상태로 업데이트해야 합니다.
- `Last Updated` 날짜를 갱신하여 문서의 신뢰성을 유지합니다.
