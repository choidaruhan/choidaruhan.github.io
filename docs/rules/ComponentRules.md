# 컴포넌트 작성 규칙 (Component Rules)

이 프로젝트는 UI 일관성과 유지보수성을 위해 Svelte 5의 최신 기능을 활용하여 개발합니다.

## 1. Svelte 5 Runes 사용

Svelte 5에서 도입된 Runes를 적극적으로 사용하여 상태 관리를 최적화합니다.

- **`$state`**: 반응형 상태를 정의할 때 사용합니다. (기존 `let` 대체)
- **`$derived`**: 다른 상태로부터 파생된 값을 정의할 때 사용합니다. (기존 `$: ` 대체)
- **`$props`**: 부모 컴포넌트로부터 전달받은 데이터를 구조 분해하여 정의합니다.

```svelte
<script lang="ts">
  let { title, content } = $props();
  let count = $state(0);
  let double = $derived(count * 2);
</script>
```

## 2. 컴포넌트 구조

- **파일 확장자**: `.svelte` 파일을 사용합니다.
- **스크립트**: `<script lang="ts">`를 기본으로 사용하며, 필요한 타입 정의를 포함합니다.
- **스타일**: 컴포넌트 내부에 지정하거나 전역 CSS 변수를 활용합니다.
- **접근성(A11y)**: 시맨틱 HTML 태그를 사용하고, 버튼이나 링크에 적절한 ARIA 레이벨을 부여합니다.

## 3. 명명 규칙

- 컴포넌트 파일 이름은 `PascalCase`를 따릅니다 (예: `PostCard.svelte`, `CustomButton.svelte`).
- 단일 책임 원칙에 따라 하나의 컴포넌트는 되도록 하나의 기능에 집중합니다.

## 4. 이벤트 처리

- 이벤트 위임이나 Svelte 5의 향상된 이벤트 핸들링 방식을 권장합니다.
- 복잡한 로직은 컴포넌트 외부의 `App Layer`나 `Shared` 유틸리티로 분리합니다.

## 목적

- 팀 내 일관된 코드 스타일을 유지하고, Svelte 5의 성능적 이점을 최대한 활용하는 개발 문화를 정착시킵니다.
