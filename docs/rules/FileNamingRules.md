# 파일 명명 규칙 (File Naming Rules)

## 1. 마크다운(.md) 파일

- 모든 마크다운 파일 이름은 첫 글자가 **대문자**로 시작하는 형식을 따릅니다. (PascalCase / Upper CamelCase)
- 예: `FileNamingRules.md`, `OneFunctionPerFileRules.md`, `DatabaseSpecification.md`

## 2. 코드 파일 (.ts, .svelte, .css 등)

- 모든 코드 관련 파일 이름은 **소문자**로 시작하는 `camelCase` 형식을 따릅니다.
- 예: `apiBase.ts`, `sidebar.svelte`, `globalStyle.css`

## 목적

- 문서 파일과 구현 파일을 파일 이름만으로도 명확히 구분할 수 있도록 합니다.
