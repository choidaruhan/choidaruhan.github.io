# Document Structure Convention

이 문서는 프로젝트 내 모든 마크다운 문서가 준수해야 할 표준 구조를 정의합니다.

## 설명

각 마크다운 파일은 반드시 다음 순서를 엄격히 준수하여 작성되어야 합니다:

1. **제목 (H1)**: 문서의 대주제를 명확히 나타내는 최상위 헤더.
2. **설명 (Description)**: 규칙, 원칙 또는 개념에 대한 상세한 기술.
3. **예시 (Examples)**: 설명을 구체적으로 뒷받침하는 코드 스니펫이나 활용 사례.

## 예시

### 준수하는 문서 구조 샘플

```markdown
# 문서 제목

## 설명
이 문서의 목적은...

## 예시
```typescript
// 예시 코드
