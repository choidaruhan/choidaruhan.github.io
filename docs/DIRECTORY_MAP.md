# DIRECTORY_MAP (디렉토리 맵)

이 문서는 프로젝트의 전체 폴더 구조와 각 디렉토리의 역할을 상세히 설명합니다.

## 📂 프로젝트 시각화 (Project Visualization)

```text
my-blog/
├── dist/               # 빌드 결과물
│   └── _app/           # SvelteKit 내부 에셋
├── docs/               # 문서 저장소
│   ├── arch/           # 아키텍처 문서
│   ├── plan/           # 계획 및 이슈
│   ├── rules/          # 프로젝트 규칙
│   └── specs/          # 기술 명세
├── scripts/            # 실행 스크립트
│   └── make/           # Makefile 헬퍼 스크립트
├── static/             # 정적 자산
└── src/                # 소스 코드
    ├── components/     # UI 공동 컴포넌트
    ├── core/           # 핵심 엔진
    │   ├── app/        # 앱 실행 로직 및 상태
    │   │   ├── boot/   # 초기화
    │   │   ├── config/ # 설정
    │   │   ├── constants/ # 상수
    │   │   └── states/ # 상태 관리
    │   ├── domain/     # 비즈니스 로직
    │   │   ├── auth/   # 인증 도메인
    │   │   └── posts/  # 게시글 도메인
    │   ├── infra/      # 기술 구현
    │   │   └── api/    # 외부 통신
    │   │       └── client/ # API 클라이언트
    │   └── shared/     # 공유 자원
    │       ├── types/  # 타입 정의
    │       │   └── ambient/ # 환경 타입
    │       └── utils/  # 공용 유틸리티
    └── routes/         # 페이지 라우팅
        ├── login/      # 로그인 페이지
        ├── post/       # 게시글 상세
        │   └── [slug]/ # 슬러그 기반 경로
        └── write/      # 글쓰기 페이지
```

## 🏗️ 폴더별 상세 설명 (Folder Descriptions)

### `/dist`
프로젝트 빌드(`npm run build`) 결과가 저장되는 곳입니다. 브라우저에서 실행 가능한 최적화된 파일들이 위치합니다.
- **`/_app`**: SvelteKit이 생성한 자바스크립트 번들과 스타일 시트가 들어있는 내부 폴더입니다.

### `/docs`
프로젝트의 지식 관리 센터입니다. 모든 중요 결정과 규칙이 이곳에 마크다운 파일로 기록됩니다.
- **`/arch`**: 프로젝트의 전체적인 설계 방향과 기술 스택을 다룹니다.
- **`/plan`**: 로드맵, 이슈 트래커 등 프로젝트의 진행 상황을 관리합니다.
- **`/rules`**: 코딩 스타일, 파일 명명, 문서 작성법 등 팀이 지켜야 할 규칙을 정의합니다.
- **`/specs`**: API 명세, DB 스키마 등 구체적인 기술 사양을 포함합니다.

### `/scripts`
개발 및 운영 효율화를 위한 자동화 도구들입니다.
- **`/make`**: `Makefile`에서 사용하는 실제 쉘 스크립트들이며, 빌드/배포/상태 확인 등의 로직을 담당합니다.

### `/static`
컴파일 과정 없이 브라우저에 그대로 노출되는 파일들입니다. 파비콘, 이미지, `robots.txt` 등이 포함됩니다.

### `/src/components`
여러 페이지에서 공통으로 재사용하는 Svelte UI 컴포넌트들입니다. 특정 도메인 로직보다는 UI 표현에 집중합니다.

### `/src/core/app`
애플리케이션의 뼈대를 구성하며 초기 구동과 전역적인 설정을 관리합니다.
- **`/boot`**: 프로젝트가 시작될 때 실행되어야 하는 초기화 함수들이 위치합니다.
- **`/config`**: 환경 변수나 전역 설정을 읽어오고 관리합니다.
- **`/constants`**: 전역적으로 쓰이는 변하지 않는 값들을 정의합니다.
- **`/states`**: Svelte 5의 Runes($state 등)를 사용하여 전역 상태를 관리합니다.

### `/src/core/domain`
블로그의 핵심 비즈니스 규칙이 정의되는 계층입니다. 외부 프레임워크나 라이브러리에 의존하지 않습니다.
- **`/auth`**: 로그인, 세션 검증, 사용자 권한 확인 등 인증 관련 핵심 로직이 위치합니다.
- **`/posts`**: 게시글 생성, 수정, 삭제 시의 데이터 규칙과 검증 절차를 다룹니다.

### `/src/core/infra`
비즈니스 로직을 구체적인 기술로 구현하는 계층입니다.
- **`/api`**: 외부 서버나 데이터베이스(Cloudflare D1)와 통신하는 코드를 포함합니다.
- **`/api/client`**: 프론트엔드 UI 계층에서 호출하기 쉽게 래핑된 API 요청 함수들이 위치합니다.

### `/src/core/shared`
도메인이나 레이어에 상관없이 유니버설하게 사용되는 자원들입니다.
- **`/types`**: 프로젝트 전체에서 준수해야 할 TypeScript 인터페이스와 타입들을 정의합니다.
- **`/types/ambient`**: 글로벌 라이브러리나 환경 선언 파일(`d.ts`)이 위치합니다.
- **`/utils`**: 날짜 포맷팅, 데이터 파싱 등 독립적인 기능을 수행하는 유틸리티 함수들입니다.

### `/src/routes`
사용자가 접근하는 URL 경로에 대응하는 페이지 컴포넌트들입니다. SvelteKit의 표준 파일 기반 라우팅을 따릅니다.
- **`/login`**: 인증을 처리하는 관리자 로그인 화면입니다.
- **`/post/[slug]`**: 게시글 주소에 따른 상세 내용을 보여주는 동적 경로입니다.
- **`/write`**: 게시물을 새로 작성하거나 기존 글을 편집하는 화면입니다.
