# Rules: 작업 관리 규칙

## 1. 파일 명명 규칙
- 모든 파일은 `NNN_상체케이스_이름.md` 형식을 따릅니다. (예: `001_AUTH_LOOP.md`)
- 번호는 생성 시점을 기준으로 순차적으로 부여합니다.

## 2. 폴더 이동 흐름
- **Issues (발견)** -> **Todo (계획 수립)** -> **History (통합)**
- 작업이 완료되면 파일을 삭제하거나 내용을 `HISTORY.md` 연대기에 통합하여 업데이트합니다.

## 3. 기록의 원칙
- 하나의 파일은 하나의 사건/기능만 다룹니다.
- `History` 파일은 나중에 보더라도 흐름을 알 수 있게 개요와 주요 변경점을 포함해야 합니다.
